# Song metadata is canonical/shared; per-user data lives in `song_user_data`

We originally deferred any shared/canonical entity for tunes (session note, since superseded), treating each user's tune as fully siloed to avoid the hard problem of deduping and reconciling metadata across users. Revisited: publishing facts (title, composer, year, lyricist) aren't copyrightable and aren't genuinely user-specific, so we decided to make Song a shared canonical entity, with a separate `song_user_data` table (per-user, private) holding each user's notes, their own display title (which may deliberately diverge from the canonical title), and the fact that the Song is in their list. Also renamed Tune to Song, since "tune" reads as jazz-specific and this app spans genres including classical. Recording identity and its separate `user_recording_data` layer were settled later in [ADR-0008](0008-provider-neutral-music-entities-and-user-data.md).

Any user can create a new Song when nothing matches on search/autocomplete — we are not gating creation behind admin approval, unlike Lead Sheets ([ADR-0002](0002-lead-sheets-admin-gated-publishing.md)), because bad/duplicate metadata is a data-quality nuisance, not a legal risk. Duplicate Songs are expected and will be merged by a Site Admin later; a "mark as duplicate" flow for admin review is a future feature, not built yet.

Creation and cross-User discovery are separate decisions. A newly created Song begins with `is_discoverable = false` and is immediately added to its creator's private Song data. Another authenticated User can find and add the same Song only after a Site Admin enables **Visible to all users**; Add Song discovery searches the shared title and never another User's private display title. Disabling discoverability stops future discovery but preserves existing memberships and private data. Shared Song facts are admin-controlled while discoverable; during the trusted-development stage, a member may edit a non-discoverable Song's shared facts under the temporary last-write-wins policy.
